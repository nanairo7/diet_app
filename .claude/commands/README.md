# カスタムコマンド一覧

Claude Code CLI（`claude` コマンド）でプロジェクトを開いているときに
`/コマンド名 引数` の形式で呼び出せるショートカットです。

## 使い方

```bash
# プロジェクトルートで Claude Code を起動
cd ~/Documents/GitHub/diet_app
claude

# チャット欄でスラッシュコマンドを入力
/new-fix https://github.com/nanairo7/diet_app/issues/XX
```

---

## コマンド一覧

### `/new-fix` — バグ修正

GitHub Issue URL またはバグの概要を渡すと、調査→修正→PR 作成まで一連の手順を実行します。

```
/new-fix https://github.com/nanairo7/diet_app/issues/30
```

```
/new-fix 通知が設定時刻より2時間遅れて届く
```

**実行される手順：**
1. `main` の最新から `fix/<バグ名>` ブランチを作成
2. 原因を調査・報告（ここで確認を取る）
3. 承認後に修正を実装
4. `flutter analyze` → `flutter test` を実行
5. `fix:` プレフィックスでコミット
6. PR を作成して URL を報告

---

### `/new-feature` — 新機能開発

機能の概要を渡すと、設計確認→実装→PR 作成まで一連の手順を実行します。

```
/new-feature 週次・月次グラフ画面をタブに追加したい
```

```
/new-feature 食事記録をCSVでエクスポートできるようにしたい
```

**実行される手順：**
1. `main` の最新から `feat/<機能名>` ブランチを作成
2. 実装方針・影響ファイルを整理して報告（ここで確認を取る）
3. 承認後に実装（文字列定数・テスト追加を含む）
4. `flutter analyze` → `flutter test` を実行
5. `feat:` プレフィックスでコミット
6. 影響する設計書（`docs/design/`）を更新
7. PR を作成して URL を報告

---

## ポイント

- どちらのコマンドも**実装前に方針確認のステップ**が入ります。
  その確認に「進めてください」と返すと実装フェーズに移ります。
- Issue URL を渡すと背景・再現手順が自動で読み込まれるため、
  口頭で説明するより精度が上がります。
- コマンドを使わず直接指示を出す場合も、同じ手順に従ってもらうよう
  「`main` から新規ブランチを作成し」と明示すると確認往復が減ります。
